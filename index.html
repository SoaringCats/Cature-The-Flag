jtWar

Work-in-Progress Team Fortress Two mimic in Minecraft

Download Download fork jtWar on GitHub

Developers are able to view further information here (http://forums.nekotech.tk/viewtopic.php?f=38&t=142)

jtWar is a simple, easy to use plugin for CraftBukkit Minecraft servers requiring no additional plugins. The plugin contains an optional flat world generator which can be enabled in bukkit.yml by setting the worldgenerator to jtWar. Due to how this plugin works it is essential for users to request changes and report bugs in the plugin to help keep it moving along and keep it fun. Issues can be easily reported on the GitHub issues page and can be used not only for issue/bug reports but also for requests.

How to setup jtWar

To setup the plugin, simply download jtWar and put it into your /plugins folder and optionally setup bukkit.yml (instructions further down) which will allow you to use the in-built flatworld generator. After you have done this, connect to the server with operator privileges (or permission node jtwar.admin) and you will be automatically teleported to 0,0 (x,y) on the map and given creative mode so you can setup the war zone. It is recommended to encase the war zone with a wall so that it is not possible for users to just run away from the action. In the war zone you are required to have a blu, red and spectator spawn. The spectator spawn will only be used by users when they first join the server so it can what you want although it is suggested that it be a top view of the action with glass blocking exit or a hidden room that contains information on how to join, etc. Please see Commands for information on how to set the spawning areas and start the game.

Commands

Once you have created the spawn points and war zone simply go to each spawn point and use the associated command. To set blu's spawn point say /blu, to set red's spawn point say /red and to set the spectator spawn point say /spectate. Once you have completed this you will be ready to start the game and may say /ready. Once the server has enabled ready mode mobs will begin to spawn, players will be allowed to /join teams and the game is in session!

Teams, Classes and Assignment Explained

When a player issues the command /join or dies they are assigned a team and class. The team assignment will be announced to the server and a team player count will be displayed. Pressing your bound key to show the Player List will allow for an easy way to find out what team you are on (quite buggy, works 99% of the time) and will show what teams the other players are on. During the game you are able to privately message your team by prefixing your message with .. For example .quickly rush the back! will tell your team-mates only the message quickly rush the back! which should allow for coordinated attacks. Furthermore, if a griefing/hacked client uses the . prefix for their hack toggles this should prevent them from proper interaction with the game. Team assignment is simple. If the blu team has less players than the red team the player will be assigned to blu team, if the red team has less players than the blu team the players will be assigned to the blu team or if the blu team has the same amount of players as the red team the player will be assigned randomly to either team. Team mates are not allowed to hurt each other and if they try the damage will not be given and the damager will be warned. After a player has been assigned to a team they will automatically be assigned a random class. Currently there are five classes in the game which are Heavy, Scout, Sniper, Pyro, and Monster. Heavy is a class that moves very slowly but is very powerful. Scout is a class which moves very fast and knocks players back when they are hit. Sniper is a class that has unlimited arrows and can take a fair bit of damage. Pyro is a class that ignites players for 1 second when they are hit and cannot be lit on fire. Monster is a class that makes loud noises when they are damaged and when they damage players, has the strongest weapons but leaves a trail of noise and fire sparks behind them when they move making them easy to spot. Class giving is random and all classes receive a set of diamond armor, enchantments and potion effects specific to their classes. Potion effects can control speed, damage resistance, etc.

Mobs

Certain mobs are allowed and disallowed in jtWar to allow for easier game play. Allowed mobs are CAVE_SPIDER, SKELETON, CREEPER, PIG_ZOMBIE and ZOMBIE. If a mob tries to spawn and is not allowed, the mob type will be automatically changed and a new mob will be spawned in its place. Mobs are included in the game to allow access to food items. Food items that will drop are COOKIE, BREAD, APPLE, PORK, COOKED_BEEF, COOKED_CHICKEN, COOKED_FISH and MELON. 'Specialty' items that drop are currently only TNT which is explained further on. Eating allows for the player to regain health letting them last longer.

TNT, Block Placement, Block Destruction

When the game is in ready mode block placement and block destruction are disallowed unless it is a TNT block placement. TNT may only be placed on top of OBSIDIAN and will ignite automatically once that code has been fixed. TNT allows you to kill players more easily and when a player is killed by a BLOCK_EXPLOSION caused by TNT they will drop lots of experience points and 10 separate TNT items.

Additional Information / Recommended Plugins

It is recommended that the plugin NoCheat (superceeded by NoCheat+, EssentialsAntiCheat, NoCheat+ is recommended ;]) is ran on the server to allow detection of cheaters by administrators.
